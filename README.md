# Бот-компаньон «Семья» (MVP, Telegram)

MVP-реализация Telegram-бота по ТЗ: онбординг, роли, меню, AI-компаньон, напоминания, SOS, базовая аналитика, а также каркасы для всех остальных модулей (социальный активатор, игры, ностальгия, мини-курсы и т.д.).

## Требования

- Python 3.11+
- Установленный `pip`

## Установка

```bash
cd "Семья"
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота_из_BotFather

# Опционально: Deepseek / другой AI
DEEPSEEK_API_URL=https://api.deepseek.example/v1/chat/completions
DEEPSEEK_API_KEY=секретный_ключ

# Опционально: погода
OPENWEATHER_API_KEY=ключ_от_OpenWeather

DEFAULT_TIMEZONE=Europe/Moscow

# Если видите TimedOut / ConnectTimeout до api.telegram.org (часто из-за сети или блокировок):
# укажите прокси (тот же, что в VPN-клиенте как HTTP-прокси, или локальный порт прокси).
TELEGRAM_PROXY=http://127.0.0.1:7890
# Альтернатива: стандартные переменные HTTPS_PROXY / ALL_PROXY (httpx их тоже понимает).

# Необязательно: увеличить ожидание соединения/ответа (секунды)
TELEGRAM_CONNECT_TIMEOUT=30
TELEGRAM_READ_TIMEOUT=30
```

Если `DEEPSEEK_API_URL` и `DEEPSEEK_API_KEY` не заданы, бот будет отвечать тёплыми заготовленными фразами без вызова внешнего AI.

Для **SOCKS5**-прокси установите зависимость: `pip install "python-telegram-bot[socks]"` и в `TELEGRAM_PROXY` укажите, например, `socks5://127.0.0.1:1080`.

## Запуск

```bash
python bot_main.py
```

Бот стартует в режиме long polling. Откройте Telegram, найдите своего бота по имени и отправьте команду `/start`.

## Что уже есть в MVP

- Онбординг:
  - Выбор роли: «Я пользователь» / «Я родственник»
  - Для пожилого: вопросы имя → возраст → город → интересы
  - Для родственника: ввод кода привязки (пока без реальной проверки)
- Главное меню с крупными кнопками и эмодзи:
  - `💬 Поговорить`
  - `📅 Напоминания`
  - `👥 События`
  - `🆘 ПОМОЩЬ`
  - `👨‍👩‍👧 Семья`
  - `⚙️ Настройки`
- AI-компаньон:
  - При любом тексте после онбординга бот отвечает через `ai_stubs.generate_companion_reply`
  - Если Deepseek не настроен — тёплый шаблонный ответ

## Основные команды

- **Онбординг и роли**
  - `/start` — запуск бота, выбор роли (пожилой / родственник) и заполнение анкеты.

- **Общий компаньон и диалог**
  - Любой текст после онбординга — разговор с AI-компаньоном.

- **Здоровье и напоминания**
  - `📅 Напоминания` — список напоминаний.
  - `/add_meds` — настроить ежедневное напоминание о приёме лекарств.
  - `/health_more` — описание расширенных возможностей напоминаний (питьевой режим, гимнастика и т.п.).

- **Погода**
  - `/weather` — утренний прогноз погоды по городу из анкеты.

- **SOS и безопасность**
  - `🆘 ПОМОЩЬ` — кнопка SOS, логируется и при наличии привязанных родственников шлёт им уведомление.
  - `/enable_checkin` — включить ежедневный вопрос «Как дела?» в 10:00.
  - `/disable_checkin` — отключить ежедневный вопрос.

- **Семейный мост и аналитика для пользователя**
  - `👨‍👩‍👧 Семья` — дневник активности за последние 24 часа (разговоры, напоминания, SOS).
  - `/add_relative <telegram_id>` — привязать родственника к пожилому пользователю.

- **Социальный активатор**
  - `👥 События` — описание модуля мероприятий.
  - `/companions` — текст о поиске компаньонов.
  - `/volunteers` — текст о волонтёрской помощи и безопасности.

- **Быт, игры, ностальгия, обучение**
  - `/helper` — помощник по дому и быту (описание).
  - `/games` — игры и викторины (описание).
  - `/nostalgia` — память и воспоминания (описание).
  - `/courses` — мини-курсы (описание).
  - `/achievements` — система достижений и геймификация (описание).

- **Голосовой интерфейс и админ-аналитика**
  - `/voice_help` — как будет работать голосовой интерфейс.
  - `/admin_stats` — описание будущей аналитики и дашбордов.

## Соответствие ТЗ

- Все крупные модули из ТЗ представлены:
  - Ключевой функционал (онбординг, роли, меню, AI-компаньон, напоминания о лекарствах, SOS, дневник активности, простая привязка родственников, погода, ежедневный чек-ин) — **реализован в коде**.
  - Остальные разделы (социальный активатор, волонтёры, игры, ностальгия, мини-курсы, достижения, голос, расширенная аналитика и B2B-сценарии) — **реализованы как дружелюбные текстовые заглушки** в `features_stub.py` и привязаны к командам в `bot_main.py`. Это позволяет постепенно наращивать реальную логику без изменения интерфейса для пользователей.
